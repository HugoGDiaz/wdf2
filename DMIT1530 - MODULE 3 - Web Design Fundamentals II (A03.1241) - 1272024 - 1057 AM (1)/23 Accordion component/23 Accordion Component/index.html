The Accordion Component

An accordion component is a user interface (UI) element that you've likely come across before. It's named after the musical instrument because segments of it will expand and collapse just like the bellows of a real accordion.

An accordion component has multiple sections or panels that can expand or collapse when clicked or tapped by the user.

With most accordion components, only one section can be expanded or collapsed at a time. This requires a little more JavaScript than we currently know, but the logic would go something like this: when the clicks on a closed section, the script checks to see if anything else is open; if there is, it closes it, then opens the selected section.

Accordion components can be a great use of space. By reducing the visual 'clutter', we can keep our users from being overwhelmed by looking at too many things at once. This is a technique called 'progressive disclosure', where information is gradually revealed as needed.

Also, remember that most users will want to skim your content, not read every word. This lets them skip over things that they may not need to read without having to endlessly scroll for the things that they're looking for.

  • We always need to make sure that anything within an accordion is accessible to all users. We can do this by using semantic elements like buttons and headings, with ARIA attributes as needed.
  • We also need to make sure that accordions behave appropriately on mobile devices. This means we need to avoid hover states and instead rely on click (or tap) events for the main functionality of the component.
  • Finally, we should always make sure that our code and animations are optimised for smooth performance. This means that we should keep our document structure as simple as possible and make sure that our users do not have to wait too long for our animations to finish before they're able to engage with the content.
